

# Изменение 1: Унифицированный хедер с навигацией

## Что делаем

Объединяем два разных хедера (страницы Инициатив и Люди) в один универсальный компонент с чёткой навигацией.

## Визуальное сравнение

**Сейчас — два разных хедера:**
```text
Страница Инициатив:
┌────────────────────────────────────────────────────────────────┐
│ ← Дашборд   [A] Админка     📊 15 из 42 инициатив   💾 Сохр.  │
└────────────────────────────────────────────────────────────────┘

Страница Люди:
┌────────────────────────────────────────────────────────────────┐
│ ← Админка   [👥] Люди       8 чел. • 5 инициатив              │
└────────────────────────────────────────────────────────────────┘
```

**Станет — единый хедер:**
```text
┌────────────────────────────────────────────────────────────────┐
│ ← Дашборд   [A] Управление                                     │
│                                                                │
│ [Инициативы|Люди]   📊 15 / 42   👥 8 чел.   💾 Сохранено  ⚙️  │
└────────────────────────────────────────────────────────────────┘
```

## Преимущества

1. **Понятная навигация** — toggle "Инициативы / Люди" сразу в хедере, на видном месте
2. **Единый брендинг** — одинаковый логотип и название "Управление"
3. **Контекст сохраняется** — статистика показывает и инициативы, и людей
4. **Меньше путаницы** — не надо думать, как вернуться или переключиться

## Технические изменения

### 1. Расширяем `AdminHeader.tsx`

Добавляем:
- Prop `currentView: 'initiatives' | 'people'` — какой раздел активен
- Prop `onViewChange` или используем `Link` для навигации
- Prop `peopleCount` — для отображения статистики людей
- Встраиваем ToggleGroup прямо в хедер (вместо ScopeSelector)

### 2. Обновляем `ScopeSelector.tsx`

Удаляем:
- Toggle "Инициативы / Люди" (переносим в хедер)

Оставляем:
- Только фильтры Unit/Team
- Summary текст

### 3. Обновляем страницы

`Admin.tsx`:
- Передаём `currentView="initiatives"` в AdminHeader

`AdminPeople.tsx`:
- Заменяем inline хедер на общий `AdminHeader`
- Передаём `currentView="people"`

## Порядок реализации

1. Модифицирую `AdminHeader.tsx` — добавляю навигационный toggle и дополнительные пропы
2. Модифицирую `ScopeSelector.tsx` — убираю toggle, оставляю только фильтры
3. Обновляю `Admin.tsx` — передаю новые пропы
4. Обновляю `AdminPeople.tsx` — заменяю inline header на общий компонент
