

# Ускорение анимаций тримапа: баланс между плавностью и отзывчивостью

## Проблема

При быстрой навигации (юнит -> команда) drilldown 1400мс создает ощущение "подвисания" интерфейса. Очередь кликов решает проблему потери ввода, но не устраняет визуальную задержку.

## Изменения

### 1. `src/components/treemap/types.ts`

| Параметр | Было | Станет |
|---|---|---|
| drilldown | 1400ms | **1200ms** |
| navigate-up | 800ms | **700ms** |
| filter | 750ms | 750ms (без изменений) |

### 2. `src/components/treemap/TreemapNode.tsx`

Exit transition: `0.35` -> `0.3` (300мс)

### 3. `src/components/treemap/TreemapContainer.tsx`

| Параметр | Было | Станет |
|---|---|---|
| Guard timeout (isAnimatingRef) | 1100ms | **900ms** |
| renderDepth delay | 900ms | **800ms** |

Guard 900мс -- пользователь может кликнуть следующий узел через ~0.9с вместо 1.1с. С учетом easing-кривой к этому моменту ~85% движения уже завершено. renderDepth delay 800мс = navigate-up (700) + 100мс буфер для завершения exit-анимаций.
